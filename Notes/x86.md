Unlike [[RISC-V Architecture|RISC-V]], **x86** is a **CISC** [[Instruction Set Architecture (ISA)|ISA]].
* It is a **stack-oriented instruction set**, so a **stack** is used to pass **arguments** to **functions**, and to save the **PC**.
* **Arithmetic instructions** can access **memory** directly, unless in **RISC-V**, which used dedicated load/store instructions.
* It heavily utilises **condition codes**, which are **flags** set based on the result of **operations**.
* We will specifically consider the **AT&T** **syntax** of **x86**.

> [!Note]
> Note that we are specifically looking at **x86-64**, which is the implementation of **x86** for **64**-[[Binary|bit]] **architectures**.
